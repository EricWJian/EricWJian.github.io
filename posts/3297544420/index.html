RuoYi项目的部署

记一次 RuoYi(v4.7.3) 项目的 部署方式。官方文档

一、RuoYi、RuoYi-Vue

RuoYi 和 RuoYi-Vue 项目结构差不多，部署方式也差不多。

1、目录

2、新加库表

新加库表，将 sql 文件下的数据导入。

3、打包服务端项目

执行 package.bat 脚本。在 ruoyi-admin 模块下 生成打包好的 jar 包。

4、redis 和 mysql

服务端首先启动 redis 和 mysql。

5、运行 服务端项目

将 ruoyi-admin.jar 和 ry.sh 文件拷贝到 linux中

出现

在 notepad++ 中 修改编码。

启动

6、前端项目打包

vscode 中打开 ruoyi-ui.

npm install:安装项目需要的 npm 包。

npm run build:prod: 打包正式环境

7、启动访问

地址：http://192.168.58.130/

二、RuoYi-Cloud

1、目录

2、修改 模块的配置文件

由于我们将 项目部署在 docker 容器中，docker容器之间访问，使用项目名，所以修改每个项目的配置文件。主要修改是 nacos 的注册中心的地址，改成以项目名进行访问。

3、项目打包

1、服务端打包

2、前端打包

在 vscode 中打开 ruoyi-ui 项目。

在 命令行工具中运行

npm install:安装项目依赖的 npm 包.

num run build:prod:打包项目；这个地方注意和 nginx.conf 的配置文件进行关联。

4、移动 jar 包

将打包好的 jar 文件移动到 docker 目录中。

执行 docker 目录下的 copy.sh 脚本。

注意：由于每个 版本的出厂 SQL 存在差异，可能 ry_config.sql没有及时修改。

修改 copy.sh

具体为 sql 文件下的 sql 脚本。

5、将项目上传到 linux 服务器。

将所有的 服务都运行在一台机器中，暂时不用分布式。

将 docker 文件整个上传到 linux 中。

6、docker 基本模块

首先启动 redis,mysql,和nacos项目(我的 容器是已将打包好的，如果第一次运行，等到运行完成就行)。

查看项目是否启动成功

ruoyi-redis:

ruoyi-mysql:

ruoyi-nacos:

7、修改注册中心配置文件

由于微服务启动的时候，读取的是配置中的配置，检查nacos配置中心的配置项，主要修改 redis 和 mysql的地址，将用 ip 地址访问修改为 通过服务名称进行访问。

8、docker 启动模块

运行模块，nacos 中查看 服务是否注册到 nacos中

9、启动访问

浏览器访问：http://192.168.58.131/

一般出现验证码，就证明可以访问了。
