1、授权码

授权码（authorization code）方式，指的是第三方应用先申请一个授权码，然后再用该码获取令牌。

1、授权服务

SpringOauth2的配制要通过继承AuthorizationServerConfigurerAdapter这个类来配制信息，并声明@EnableAuthorizationServer注解，表示这是认证服务。

AuthorizationServerConfigurerAdapter 中:

  • ClientDetailsServiceConfigurer：用来配置客户端详情服务（ClientDetailsService），客户端详情信息在这里进行初始化，你能够把客户端详情信息写死在这里或者是通过数据库来存储调取详情信息。
  • AuthorizationServerSecurityConfigurer：用来配置令牌端点(Token Endpoint)的安全约束。
  • AuthorizationServerEndpointsConfigurer：用来配置授权（authorization）以及令牌（token）的访问端点和令牌服务(token services)。

AuthorizationServerEndpointsConfigurer 中：

AuthorizationServerEndpointsConfigurer 是 Spring Security OAuth2 框架中的一个配置类，用于配置授权服务器的端点（Endpoint）。

授权服务器的端点是指可以接收客户端请求的 URL 地址，如获取授权码的 /oauth/authorize、获取访问令牌的 /oauth/token 等。AuthorizationServerEndpointsConfigurer 可以配置以下内容：

  • 认证管理器（AuthenticationManager）：用于处理客户端的身份验证请求。
  • 用户详情服务（UserDetailsService）：用于获取用户的详细信息，包括用户名、密码、角色等。
  • 访问令牌转换器（AccessTokenConverter）：用于将访问令牌转换为不同格式的数据，如 JWT（JSON Web Token）。
  • 令牌存储（TokenStore）：用于存储和管理访问令牌和刷新令牌。
  • 授权码存储（AuthorizationCodeServices）：用于存储和管理授权码。
  • 回调 URL 处理器（RedirectResolver）：用于处理客户端回调 URL，使其与授权服务器的 URL 匹配。
  • 用户授权处理器（UserApprovalHandler）：用于处理用户授权请求，包括批准或拒绝请求。

通过配置 AuthorizationServerEndpointsConfigurer，可以自定义授权服务器的端点处理逻辑，以满足不同的业务需求。

2、简化模式，密码模式，客户端模式

  • 简化模式（implicit）和授权码模式相比，取消了获取授权码的过程，直接获取访问令牌；

  • 密码模式（password）客户端直接向用户获取用户名和密码，之后向认证服务器获取访问令牌；

  • 客户端模式（client credentials）客户端直接通过客户端认证（比如client_id和client_secret）从认证服务器获取访问令牌。
