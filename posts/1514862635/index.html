链表

一、数组的优点
随机访问性强，查找速度快

二、数组的缺点
1、对内存的要求高，必须要连续的内存空间
2、插入和删除的效率比较低
3、数组大小固定，动态拓展性差，而且可能浪费内存

三、链表的优点
1、对内存的要求低
2、大小可以不固定，内存的利用率比较高
3、插入和删除方便

四、链表的缺点
查找效率低，不可以随机查找，每次查找必须从头开始

单链表的特点

  1. 链表增删元素的时间复杂度为O(1),查找一个元素的时间复杂度为 O(n);
  2. 单链表不用像数组那样预先分配存储空间的大小，避免了空间浪费
  3. 单链表不能进行回溯操作，如：只知道链表的头节点的时候无法快读快速链表的倒数第几个节点的值。

单链表的创建和销毁

创建

  1. 链表是以节点的方式来链式存储的。
  2. 每个节点包含 data 域，next域，next域指向下一个节点。
  3. 链表的各个节点不一定是连续存储的。
  4. 链表分 带头节点的链表和没有头节点的链表。

求单链表的中间元素

快慢指针法：假如我们设置 两个指针 slow、fast 起始都指向单链表的头节点。其中 fast 的移动速度是 slow 的2倍。当 fast 指向末尾节点的时候，slow 正好就在中间了。

判断一个链表是否是是循环链表

快慢指针：假如我们设置 两个指针 slow、fast 起始都指向单链表的头节点。其中 fast 每次移动两个节点，slow 每次移动1个节点。当某一时刻 fast 追上 slow 的时候，就证明存在环。

已知一个单链表求倒数第 N 个节点

解法跟快慢指针类似，滑动窗口法，我们可以想一下：如果我们让快指针先走 n-1 步后，然后让慢指针出发。快慢指针每次都只移动一个位置，当快指针移动到链表末尾的时候，慢指针是否就正处于倒数第 N 个节点的位置。

删除单链表的倒数第 n 个节点

也是采用滑动窗口的方法。

树

1、 有且只有一个节点没有父节点：该节点称为树的根。

二叉树

二叉树在满足树的条件下增加以下的限制条件：

  1. 每个节点最多有两个孩子。
  2. 这两个节点有左右之分，次序不可颠倒。

面试题：

  1. 二叉树的深度。
    • 层次遍历二叉树，最大层次即为二叉树的高度。

完全二叉树

完全二叉树是特殊的二叉树：

  1. 除了最后一层，每一层都是填满的。
  2. 最后一层所有的节点都是靠左排列。

满二叉树

满二叉树是特殊的完全二叉树。

  • 满二叉树的总节点数：2^h-1。h是树的高度。

  • 满二叉树的高度：log2(n+1) 。n是树中节点的数量。

大顶推

  • 各个父节点的值总是大于或等于任何一个子节点的值。

  • 最大元素刚好存储在堆顶。

小顶堆

  • 各个父节点的值总是小于或等于任何一个子节点的值。
  • 最小元素刚好存储在堆顶。

二叉搜索树

Binary Search Tree：BST

也叫二叉排序树，二叉查找树。

中序遍历二叉查找树，就能等到从小到大的序列，所以也称为二叉排序树。

存在弊端：左右节点的高度可能相差很大

AVL树

又称为 平衡二叉树。

AVL树是在二叉搜索树的基础上，增加了一条规则：对应的左右节点的深度差不能超过1。超过1会失衡，需要通过旋转操作来保持平衡。

弊端：

  1. 严格的平衡条件
  2. 频繁的插入和删除操作，旋转次数会变多。影响增加删除效率。

红黑树是什么

在AVL树基础上：

  1. 增加了四条规则
  2. 变更了一条规则

也就是红黑树的五个特征：

  1. 结点是黑色或红色
  2. 根结点是黑色
  3. 所有叶子结点都是黑色，(都是空结点、NULL/NIL结点)
  4. 根结点到叶子结点的一条路径上不能有连续的红色结点
  5. 从任意结点到其叶子结点，所有的路径上都包含相同数目的黑色结点（这一条是修改的，原来的是：AVL树规则，对应的左右节点深度差不能超过1）。

红黑树出现的目的：利用红黑亮色的变换，去替代一部分旋转的操作，来减少旋转带来的开销。

因为旋转操作比变色步骤多，而且旋转可能会影响树的平衡，导致需要再次进行旋转。

红黑树需要使用旋转和染色来保持平衡。

红黑树左旋和右旋

旋转的性质

左旋：左子树高度+1，右子树高度-1；

右旋：右子树高度+1，左子树高度-1；

树与二叉树的转换

树转换为二叉树

（1）加线：在兄弟之间加一连线。

（2）抹线：对每个结点，除了其左孩子外，抹掉其与其余孩子之间的连线。

（3）旋转：将树作适当的旋转即可。

二叉树转换为树

（1）逆旋转：把二叉树从左上到右下分为若干层。如A是一层，B-C-D是一层，E-F-G-H-I-J是一层，然后调整为水平方向。（每遇到左孩子则为一层）

（2）加线：找到每一层结点在其上一层的父结点。如第三层中，E-F的父结点是B，G的父结点是C，H-I-J的父结点是D；在第二层中，B-C-D的父结点是A，将每一层的结点和其父结点相连。

（3）抹线：删除每一层结点之间的连接。

二叉树的遍历：

广度优先遍历

  • 层序遍历
    1. 从上至下
    2. 从左到右

深度优先遍历

  • 先序遍历
    1. 先输出父节点，再遍历左子树和右子树。
  • 中序遍历
    1. 先遍历左子树，再输出父节点，最后遍历右子树。
  • 后续遍历
    1. 后序遍历：先遍历左子树，再遍历右子树，最后输出父节点。

树的遍历

数的遍历一般只有：先序遍历和后续遍历没有中序遍历。

结论

如果一个二叉树是由一个树转换而来，那么：

  1. 那对这颗二叉树树进行先序遍历就等同于对原来的树进行先序遍历。
  2. 那对这颗二叉树树进行中序遍历就等同于对原来的树进行后序遍历。

线索二叉树：

  1. 利用二叉链表的控制针域，存放指向该结点在某种遍历次序下的前驱和后继结点的指针(这种附加的指针称为”线索”)。
  2. 这种加上了线索的二叉链表称为线索链表，相应的二叉树称为线索二叉树(Threaded BinaryTree)。根据线索性质的不同，线索二叉树可分为：前序线索二叉树、中序线索二叉树和后序线索二叉树三种。

Btree和B+Tree

https://blog.csdn.net/u200814342A/article/details/132589522

B树和B+树的区别

  1. 节点结构：B树的节点包含键和对应的子节点，而B+树的中间节点只包含键和子节点指针，数据只存在于叶子节点。
  2. 叶子节点：在B树中，叶子节点既包含键也包含对应的数据，而B+树的叶子节点只包含键和数据指针，数据本身存储在叶子节点的链表中。
  3. 范围查询：由于B+树的叶子节点形成了一个有序的链表，所以B+树更适合范围查询操作，而B树的范围查询需要额外的遍历叶子节点。
  4. 磁盘访问：B+树的叶子节点形成了一个连续的链表，磁盘访问更加连续和顺序化，相比之下，B树的磁盘访问更加零散。
  5. 索引效率：由于B+树的数据只存在于叶子节点，所以每个节点能存储更多的键和子节点指针，相对于B树，B+树的高度更低，索引效率更高。

总的来说，B+树相比于B树在范围查询、磁盘访问和索引效率等方面具有优势。因此，在实际应用中，B+树更为常用，特别是在数据库系统等需要高效执行范围查询的场景中。

排序算法

内存排序

  1. 直接插入排序

    • 基本思想：把 n 个待排序元素看成一个有序表和一个无序表，开始时有序表只包含一个元素，无序表中包含 n-1 个元素，
    • 排序过程中每次从无序表取出第一个元素，把他的排序码依次与有序元素的排序码进行比较，将它插入到有序表中适当的位置，使之成为新的有序表。

  2. 希尔排序

    • 希尔排序是把记录按下标的一定增量分组，对每组使用直接插入排序算法排序;随着增量逐渐减少，每组包含为关键词越来越多，当增量减至1 时，整个文件恰被分成一组，算法便终止。
    • 希尔排序法又称缩小增量法

  3. 选择排序

    • 第一次从 arr[0]~arr[n-1] 中选取最小值，与arr[0]交换，
    • 第二次从 arr[1]~arr[n-1]中选择最小值，与arr[1]交换，
    • ……
    • 第n-1次从 arr[n-2]~arr[n-1]中选择最小值，与arr[n-2]交换，
    • 总共 通过 n-1 次，得到一个按排序码从小到大排列的有序序列。

  4. 堆排序

    • 堆排序是指利用堆（数据结构）进行选择数据的一种排序算法
    • 先将原数组建成堆，需要注意的是排升序要建大堆，排降序建小堆
    • 将待排序序列构造成一个大顶堆。此时，整个序列的最大值就是堆顶的根节点。
    • 将其与末尾元素进行交换，此时末尾就为最大值。
    • 然后将剩余n-1 个元素重新构造成一个堆，这样会得到 n个元素的次小值。
    • 如此反复执行，便能得到一个有序序列了。

  5. 冒泡排序

    • 依次比较相邻元素的值，如发现逆序则交换。使得较大的元素从前移向后端。

  6. 快速排序

    • 快速排序(Quicksort)是对冒泡排序的一种改进。
    • 基本思想是: 通过一趟排序将要排序的数据分割成独立的两部分，其中一部分的所有数据都比另外一部分的所有数据都要小，
    • 然后再按此方法对这两部分数据分别进行快速排序，整个排序过程可以递归进行，以此达到整个数据变成有序序列

  7. 归并排序

    • 将已有序的子序列合并，得到完全有序的序列；即先使每个子序列有序，再使子序列段间有序

  8. 基数排序
