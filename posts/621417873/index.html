Validation 数据校验

  1. Java API规范(JSR303)定义了 Bean 校验的标准 validation-api，但没有提供实现。
  2. hibernate validation是对这个规范的实现，并增加了校验注解如@Email、@Length等。
  3. Spring Validation 是对 hibernate validation的二次封装，用于支持 spring mvc 参数自动校验。

一、Valid 和 @Validated 比较

@Valid 和 @Validated 两个注解比较：

  1. @Valid 和 @Validated 两者都可以对数据进行校验，待校验字段上打的规则注解（@NotNull, @NotEmpty等）都可以对 @Valid 和 @Validated 生效。
  2. @Valid 支持嵌套校验。
  3. @Valid 进行校验的时候，需要用 BindingResult 来做一个校验结果接收。当校验不通过的时候，如果手动不 return ，则并不会阻止程序的执行。
  4. @Validated 进行校验的时候，当校验不通过的时候，程序会抛出400异常，阻止方法中的代码执行，这时需要再写一个全局校验异常捕获处理类，然后返回校验提示。
  5. 总体来说，@Validated 使用起来要比 @Valid 方便一些，它可以帮我们节省一定的代码，并且使得方法看上去更加的简洁。

二、常用属性

1、空和非空

  1. @NotBlank：只能作用在字符串上，字符串不能为null，而且调用 trim() 后，长度必须大于0。
  2. @NotEmpty：集合对象的元素不为 0 ，即集合不为空，也可以用于字符串不为 null 。
  3. @NotNull：被注释的元素不能为 null，可以为空字符串。
  4. @Null：限制只能为 null。
  5. 字符串“”不等于null。

2、数值检查

  1. @Max(value)：被注释的元素必须是一个数字，其值必须小于等于该值。
  2. @DecimalMax(value)：被注释的元素必须是一个数字，其值必须小于等于指定的最大值。
  3. @Min(value)：被注释的元素必须是一个数字，其值必须大于等于该值。
  4. @DecimalMin(value)：被注释的元素必须是一个数字，其值必须大于等于指定的最小值。
  5. @Digits(integer,fraction)：被注释的元素必须是一个数字，其值必须在可接受的范围内。
  6. @Negative：限制必须为负整数。
  7. @NegativeOrZero：限制必须为负整数或零。
  8. @Positiv：限制必须为正整数。
  9. @PositiveOrZero 限制必须为正整数或零。

3、Boolean 值检查

  1. @AssertTrue：被注释的元素必须为true。
  2. @AssertFalse：被注释的元素必须为false。

4、长度检查

  1. @Size(max,min)：检查该字段的 size 是否在 min 和 max 之间，可以是字符串、数组、集合、Map 等。

5、日期检查

  1. @Past：限制日期为过去的日期。

  2. @PastOrPresent：限制日期为当前或过去的日期。

  3. @Future：限制日期为将来的日期。

  4. @FutureOrPresent：限制日期为当前或将来的日期。

6、其他检查

1、 @Pattern(value)：限制必须符合指定的正则表达式。

2、@Email：限制必须为email格式。

7、 Hibernate Validator 附加的约束注解

  1. @Length(min=, max=) ：被注释的字符串的大小必须在指定的范围内。
  2. @Range(min=, max=) ：被注释的元素必须在合适的范围内。
