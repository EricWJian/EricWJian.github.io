1、全文检索

全文检索是计算机程序通过扫描文章中的每一个词，对每一个词建立一个索引，指明该词在文章中出现的次数和位置。当用户查询时根据建立的索引查找，类似于通过字典的检索字表查字的过程。

索: 建立索引 文本—->切分 —> 词 文章出现过 出现多少次

检索: 查询 关键词—> 索引中–> 符合条件文章 相关度排序

全文检索(Full-Text Retrieval)以文本作为检索对象，找出含有指定词汇的文本。全面、准确和快速是衡量全文检索系统的关键指标。

  • 只处理文本、不处理语义
  • 搜索时英文不区分大小写
  • 结果列表有相关度排序

2、简介

1、什么是ElasticSearch

​ ElasticSearch 简称 ES ，是基于Apache Lucene构建的开源搜索引擎，是当前最流行的企业级搜索引擎。Lucene本身就可以被认为迄今为止性能最好的一款开源搜索引擎工具包，但是lucene的API相对复杂，需要深厚的搜索理论。很难集成到实际的应用中去。ES是采用java语言编写，提供了简单易用的RestFul API，开发者可以使用其简单的RestFul API，开发相关的搜索功能，从而避免lucene的复杂性。

2、ElasticSearch诞生

多年前，一个叫做Shay Banon的刚结婚不久的失业开发者，由于妻子要去伦敦学习厨师，他便跟着也去了。在他找工作的过程中，为了给妻子构建一个食谱的搜索引擎，他开始构建一个早期版本的Lucene。

直接基于Lucene工作会比较困难，所以Shay开始抽象Lucene代码以便Java程序员可以在应用中添加搜索功能。他发布了他的第一个开源项目，叫做“Compass”。

后来Shay找到一份工作，这份工作处在高性能和内存数据网格的分布式环境中，因此高性能的、实时的、分布式的搜索引擎也是理所当然需要的。然后他决定重写Compass库使其成为一个独立的服务叫做Elasticsearch。

第一个公开版本出现在2010年2月，在那之后Elasticsearch已经成为Github上最受欢迎的项目之一，代码贡献者超过300人。一家主营Elasticsearch的公司就此成立，他们一边提供商业支持一边开发新功能，不过Elasticsearch将永远开源且对所有人可用。

Shay的妻子依旧等待着她的食谱搜索……

目前国内大厂几乎无一不用Elasticsearch，阿里，腾讯，京东，美团 等等 …..

4、核心概念

1、索引 <Index>

一个索引就是一个拥有几分相似特征的文档的集合。比如说，你可以有一个商品数据的索引，一个订单数据的索引，还有一个用户数据的索引。一个索引由一个名字来标识(必须全部是小写字母的)，并且当我们要对这个索引中的文档进行索引、搜索、更新和删除的时候，都要使用到这个名字。

2、Type 文档类型(过时)

类似于mysql数据库中的table表，es中可以在Index中建立type（table），通过mapping进行映射。一个index可能会被划分为多个type, 例如: 商品中有电子商品, 服装商品, 生鲜商品…

3、映射<Mapping>

为了能够将时间域视为时间，数字域视为数字，字符串域视为全文或精确值字符串， Elasticsearch 需要知道每个域中数据的类型。这个信息包含在映射中。

**映射是定义一个文档和它所包含的字段如何被存储和索引的过程**。在默认配置下，ES可以根据插入的数据自动地创建mapping，也可以手动创建mapping。

3、文档<Document>

**文档是索引中存储的一条条数据。一条文档是一个可被索引的最小单元**。ES中的文档采用了轻量级的JSON格式数据来表示。

5、元字段解释

  • _index：文档所属的索引

    _index标注document属于哪个index,

  • _type：文档的类型

    _type元字段用来标注document属于哪个类型, 也被称作映射类型，默认创建的 类型是 _doc.

    创建 type类型为 employee，索引名称为 megacorp 的文档

  • _id：文档的id

    _id代表document的唯一标识, 与_index和_type一起, 唯一标识和定位一个document.

  • _source：文档原始JSON内容

8、索引原理

1、倒排索引

倒排索引（Inverted Index）也叫反向索引，有反向索引必有正向索引。通俗地来讲，正向索引是通过key找value，反向索引则是通过value找key。ES底层在检索时底层使用的就是倒排索引。

2、索引模型

现有索引和映射如下:

先录入如下数据，有三个字段title、price、description等

_id title price description
1 蓝月亮洗衣液 19.9 蓝月亮洗衣液很高效
2 iphone13 19.9 很不错的手机
3 小浣熊干脆面 1.5 小浣熊很好吃

在ES中除了text类型分词，其他类型不分词，因此根据不同字段创建索引如下：

  • title字段:

    term _id(文档id)
    蓝月亮洗衣液 1
    iphone13 2
    小浣熊干脆面 3

  • price字段

    term _id(文档id)
    19.9 [1,2]
    1.5 3

  • description字段

    term _id term _id term _id
    蓝 1 不 2 小 3
    月 1 错 2 浣 3
    亮 1 的 2 熊 3
    洗 1 手 2 好 3
    衣 1 机 2 吃 3
    液 1
    很 [1:1:9,2:1:6,3:1:6]
    高 1
    效 1

注意: Elasticsearch分别为每个字段都建立了一个倒排索引。因此查询时查询字段的term,就能知道文档ID，就能快速找到文档。

9、分词器

1、内置分词器

  1. Standard：标准分析器(默认分词器)，过滤符号标点，英文按单词词切分，并小写处理。
  2. Simple：简单分析器，按照单词切分，过滤符号,字母小写处理。
  3. Whitespace：空格分析器，按照空格切分，不转小写。
  4. Stop：过滤标点符号,数字，过滤语气词，停顿词(the,a,is)。
  5. Keyword：不分词，直接将输入当作输出。

2、内置分词器测试

  • 标准分词器
    • 特点: 按照单词分词，英文统一转为小写，过滤标点符号，中文单字分词。
  • Simple 分词器
    • 特点: 英文按照单词分词，英文统一转为小写，去掉符号，中文按照空格进行分词。
  • Whitespace 分词器
    • 特点: 中文、英文按照空格分词，英文不会转为小写，不去掉标点符号。

定义扩展词典和停用词典可以修改IK分词器中config目录中IKAnalyzer.cfg.xml这个文件。

注意: 词典的编码必须为UTF-8，否则无法生效!

6、高级查询

在 Spring Data Elasticsearch 中，SearchQuery 是一个接口，它定义了执行 Elasticsearch 搜索查询的各种参数。SearchQuery 有多个实现类，每个实现类都提供了不同的方式来构建 Elasticsearch 搜索查询。

以下是一些常用的 SearchQuery 实现类：

  1. NativeSearchQuery - 通过 Elasticsearch 原生查询 DSL 构建查询语句的实现类。它可以直接使用 Elasticsearch 的查询语法来构建查询语句。
  2. StringQuery - 将查询语句表示为字符串的实现类。它可以使用字符串来表示查询语句，类似于 Kibana 或其他 Elasticsearch 客户端工具。
  3. CriteriaQuery - 使用 Spring Data 的 Criteria API 构建查询语句的实现类。它可以使用 Spring Data 的 Criteria API 来构建类型安全的查询语句，但是不支持所有 Elasticsearch 的查询语法。
  4. MethodQuery - 将查询语句表示为方法名称的实现类。它可以使用 Spring Data 的方法命名规则来构建查询语句，类似于 JPA 或 Spring Data MongoDB。

除了这些常用的实现类之外，您还可以创建自己的 SearchQuery 实现类来满足特定的需求。无论使用哪种实现类，都可以通过 ElasticsearchRestTemplate 的 search 方法执行 Elasticsearch 搜索查询。

NativeSearchQuery 是用于构建 Elasticsearch 原生查询语句的查询参数对象，它提供了许多用于指定查询条件、分页信息和排序信息等的方法。

以下是一些常用的查询参数：

  1. withQuery(QueryBuilder query) - 设置查询条件，可以使用 Elasticsearch 的查询语法，例如 matchQuery、termQuery、rangeQuery 等等。
  2. withFilter(QueryBuilder filter) - 设置查询过滤条件，它和 withQuery 方法的区别在于，过滤条件不会影响查询结果的评分。
  3. withPageable(Pageable pageable) - 设置查询结果的分页信息。
  4. withSort(Sort sort) - 设置查询结果的排序方式。
  5. withIndices(String... indices) - 设置要查询的索引名称。
  6. withTypes(String... types) - 设置要查询的文档类型。
  7. withRouting(String routing) - 设置查询路由。
  8. withTrackScores(boolean trackScores) - 是否返回查询结果的评分。
  9. withTrackTotalHits(boolean trackTotalHits) - 是否返回查询结果的总数。
  10. withMinScore(float minScore) - 设置最小的评分值，只有评分高于此值的文档才会被返回。

以上仅列举了一部分常用的查询参数，还有许多其他的参数可以使用，例如 withPostFilter、withSearchType、withHighlightFields 等等，您可以查看 NativeSearchQueryBuilder 类的源代码，以了解更多关于构建 Elasticsearch 原生查询语句的方法和用法。
