1、Spring Cloud Gateway 配置说明

1、路由 Route

由ID、目标URI、断言集合和过滤器集合组成。如果聚合断言结果为真，则转发到该路由。

  1. ID：自定义的路由 ID，保持唯一
  2. URL：目标服务地址

2、断言 Predicate

路由转发的判断条件，允许开发人员匹配 HTTP 请求中的任何内容，比如请求头或请求参数，最后根据匹配结果返回一个布尔值。

3、过滤器 Filter

过滤器是路由转发请求时所经过的过滤逻辑，可用于修改请求、响应内容。

2、路由 Route

Route 主要由 路由id、目标uri、断言集合和过滤器集合组成，那我们简单看看这些属性到底有什么作用。

  • id：路由标识，要求唯一，名称任意（默认值 uuid，一般不用，需要自定义）。
  • uri：请求最终被转发到的目标地址。
  • order： 路由优先级，数字越小，优先级越高。
  • predicates：断言数组，即判断条件，如果返回值是boolean，则转发请求到 uri 属性指定的服务中。
  • filters：过滤器数组，在请求传递过程中，对请求做一些修改。

3、动态路由实现

可能存在 服务找不到的情况：

  1. 网关服务需要 引入 spring-cloud-starter-loadbalancer (高版本的 cloud)。
  2. 被调用服务需要开启 @EnableDiscoveryClient

  • Between Route Predicate Factory (请求时问满足配高时间之间)
